/hurd/crash

 
Usage: crash [OPTION...]
Server to handle crashing tasks and dump core files or equivalent.

 These options specify the disposition of a crashing process:
  -a, --action=ACTION        Action taken on crashing processes
  -O, --orphan-action=ACTION Action taken on crashing orphans

 These options are synonyms for --action=OPTION:
  -c, --core-file, --dump-core   Dump a core file
  -k, --kill                 Kill the process
  -s, --suspend              Suspend the process

  -?, --help                 Give this help list
      --usage                Give a short usage message
  -V, --version              Print program version

Mandatory or optional arguments to long options are also mandatory or optional
for any corresponding short options.

The ACTION values can be `suspend', `kill', or `core-file'.

If `--orphan-action' is not specified, the `--action' value is used for
orphans.  The default is `--action=suspend --orphan-action=core-file'.

/hurd/fakeroot

 
Usage: fakeroot [OPTION...]
A translator for faking privileged access to an underlying filesystem.

  -?, --help                 Give this help list
      --usage                Give a short usage message
  -V, --version              Print program version

This translator appears to give transparent access to the underlying directory
node.  However, all accesses are made using the credentials of the translator
regardless of the client and the translator fakes success for chown and chmod
operations that only root could actually do, reporting the faked IDs and modes
in later stat calls, and allows any user to open nodes regardless of
permissions as is done for root.

/hurd/firmlink

 
Usage: firmlink [OPTION...] TARGET
A translator for firmlinks.

  -?, --help                 Give this help list
      --usage                Give a short usage message
  -V, --version              Print program version

A firmlink is sort of half-way between a symbolic link and a hard link:

Like a symbolic link, it is `by name', and contains no actual reference to the
target.  However, the lookup returns a node which will redirect parent lookups
so that attempts to find the cwd that go through the link will reflect the link
name, not the target name.  The target referenced by the firmlink is looked up
in the namespace of the translator, not the client.
